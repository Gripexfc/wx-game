# 目标选择与动态经验设计文档

**日期：** 2026-04-21
**主题：** 首次目标选择、首页承诺卡样式升级、任务经验动态计算
**核心理念：** 让“选目标”和“坚持目标”都更有陪伴感，让经验值真正体现坚持时间和噜噜心情

---

## 一、背景与目标

当前实现存在以下问题：

- 首次进入只能通过较弱的系统菜单选择推荐目标，自定义目标入口不够直接
- 首页 `+ 添加目标` 仅为普通虚线占位，和当前暖系 UI 语言不统一，缺少识别度
- 目标经验主要由固定值或简单倍率决定，没有体现“坚持时间”和“心情值”的正相关
- 用户无法直观看到目标为什么值这些经验，也无法感知坚持天数对成长的帮助

**设计目标：**

- 首次进入时就能明确选择“推荐目标”或“自定义目标”
- 首页空承诺槽位沿用同一套目标选择体验，避免入口割裂
- 目标卡片更符合当前 UI 设计，具有明确引导感和完成反馈
- 经验值改为“基础经验 + 坚持加成 + 心情加成 + 心愿奖励”的可解释模型

---

## 二、范围与结论

本次按用户确认的 `两处都改 + 完整目标系统升级方案` 执行：

1. **首次进入引导升级**
   - 起名完成后进入目标选择步骤
   - 同时展示推荐目标和自定义目标入口

2. **首页空槽位升级**
   - 4 个 `添加目标` 槽位改为更有设计感的卡片入口
   - 点击后复用与首次进入一致的选择逻辑

3. **目标经验规则升级**
   - 最终 XP 与目标基础经验、连续坚持天数、噜噜当前心情正相关
   - 保留心愿额外奖励，但与主公式拆开展示

4. **数据层补强**
   - 目标补充 streak、来源、基础经验等字段
   - 推荐目标和自定义目标统一走同一套创建逻辑

### 2.1 关键决策

本次实现以“体验更顺、视觉更统一”为优先，不考虑旧逻辑兼容。

- **不保留旧弹窗流程**：目标选择不再以 `wx.showActionSheet` 为主承载
- **不保留旧字段语义**：`goal.xp` 不再作为核心字段，统一改为 `baseXp + 实时计算`
- **不保留旧入口优先级**：`TaskPage` 不再参与主流程设计
- **不为旧数据迁移让步**：如果旧结构阻碍实现，允许直接调整现有数据模型和调用链

### 2.2 UI 优先原则

本次 UI 设计以“好看、统一、符合当前暖系设计”为优先级，高于对现有原生弹窗和旧页面结构的保守复用。

具体原则：

- 优先选择 **Canvas 内统一设计的面板 / 覆盖层**
- 避免使用系统原生 `ActionSheet` 作为主要视觉界面
- 允许为了视觉一致性新增 UI 状态与组件
- 允许直接替换当前较弱的空槽位样式和首次引导流程

## 三、核心用户流程

### 3.1 首次进入流程

```text
输入名字
  ↓
进入目标选择步骤
  ↓
可选：推荐目标 / 自定义目标
  ↓
创建目标并加入今日承诺
  ↓
生成或刷新今日心愿
  ↓
返回首页，看到已填入的承诺卡
```

**决策：首次目标选择为必选步骤。**

- 不再保留 `稍后再说`
- 用户至少选定 1 个推荐目标或创建 1 个自定义目标后，才进入首页
- 这样能保证首次进入就建立“目标陪伴感”和完整的视觉体验

### 3.2 首页添加流程

```text
点击首页空承诺槽位
  ↓
打开目标选择面板
  ↓
选择推荐目标或创建自定义目标
  ↓
加入第一个空槽位
  ↓
保存数据并给予噜噜反馈
```

### 3.3 统一入口原则

首次进入与首页空槽位虽然入口位置不同，但后续逻辑保持一致：

- 使用相同的推荐目标数据源
- 使用相同的自定义目标输入方式
- 使用相同的目标创建与承诺写入逻辑
- 使用相同的宠物反馈与数据保存流程

这样可以避免两套近似逻辑长期分叉。

### 3.4 交互承载方式

**目标选择面板明确采用 Canvas 内部的全屏选择页 / 覆盖层承载，不使用原生 `ActionSheet` 作为主界面。**

实现方向：

- 首次进入：`OnboardingPage` 在名字确认后切换到目标选择子状态
- 首页空槽位：`HomePage` 打开全屏覆盖层或页内覆盖面板
- 两处共用同一套推荐卡、自定义卡、按钮和选中反馈绘制逻辑

这样可以确保：

- UI 风格统一
- 推荐卡片、自定义卡片和预计经验展示真正可视化
- 后续动效和选中态不受系统弹窗限制

## 四、UI 设计

### 4.1 整体风格

延续当前首页的暖系、奶油感、陪伴型风格，不引入新的高对比设计语言。

**关键词：**

- 温暖
- 有陪伴感
- 有目标感
- 比当前更精致，但不过度装饰

### 4.2 首页空槽位改造

当前空槽位由普通虚线框升级为“可点击的目标邀请卡”。

**视觉特征：**

- 浅暖黄卡底
- 柔和边框和轻微高光
- 中心图标使用 `🎯` 或 `✨`
- 主文案：`添加今日目标`
- 副文案：`可选推荐，也可自定义`

**交互反馈：**

- 点击时保留当前按压缩放节奏
- 添加成功后，新目标卡比普通卡略亮，形成“刚放入”的确认感

### 4.3 推荐目标卡

在目标选择面板中，推荐目标使用列表卡片展示，而不是仅展示系统 `ActionSheet` 文本。

每张推荐卡包含：

- 图标
- 目标名称
- 目标标签或推荐理由
- 预计经验范围

**推荐理由示例：**

- 适合新手开始
- 更容易坚持
- 对心情更友好
- 今天就能完成

### 4.4 自定义目标卡

自定义目标不再藏在最后一个菜单项中，而是作为与推荐目标同级的卡片：

- 标题：`自己写一个`
- 副文案：`更适合你今天的节奏`
- 点击后再进入输入

这样首次进入就能明确看到“推荐 / 自定义”两个选择方向。

### 4.5 已添加目标卡增强

首页承诺卡在现有基础上补充更能解释经验来源的信息：

- 目标名
- 图标
- 完成态或可得经验
- 小标签：`第 N 天`
- 小标签：`心情加成`

不直接在首页堆完整公式，只展示足够理解的结果。

### 4.6 面板布局决策

目标选择面板按“好看优先”的方式设计为完整视觉页面，而不是简化弹窗：

- 顶部标题：`今天想和噜噜坚持什么？`
- 中部为推荐目标卡片区，2 列排布
- 底部保留一张更醒目的 `自己写一个` 卡片
- 卡片支持选中高亮、轻微缩放和暖色边框反馈
- 首次进入与首页添加都使用同一套布局，只调整标题文案

## 五、经验公式设计

### 5.1 核心公式

完成目标时的经验改为：

```text
最终 XP = 基础经验 × 坚持系数 × 心情系数 + 额外奖励
```

其中：

- **基础经验**：目标定义中的 `baseXp`
- **坚持系数**：由该目标连续完成天数决定
- **心情系数**：由当前噜噜心情值决定
- **额外奖励**：当天目标若同时命中心愿，则追加心愿奖励

### 5.2 坚持系数

| 连续完成天数 | 系数 |
|---|---|
| 第 1 天 | 1.0 |
| 2-3 天 | 1.1 |
| 4-6 天 | 1.2 |
| 7-13 天 | 1.35 |
| 14 天以上 | 1.5 |

### 5.3 心情系数

| 心情值 | 系数 |
|---|---|
| 0-39 | 0.85 |
| 40-59 | 0.95 |
| 60-79 | 1.0 |
| 80-89 | 1.1 |
| 90-100 | 1.2 |

### 5.4 上限控制

- 单次目标经验上限：**40 XP**
- 若计算结果异常或非数字，降级回 `baseXp`

### 5.5 用户可见说明

用户无需在首页看到完整公式，但完成反馈中需能理解来源：

```text
基础 15
坚持加成 +3
心情加成 +2
心愿奖励 +5
本次共 +25 XP
```

### 5.6 适用范围

为避免规则过度复杂，本次“坚持系数”只作用于**可重复坚持的目标**：

- `habit`
- 自定义持续型目标

以下目标不参与 streak 系数，默认按 `1.0` 计算：

- `oneTime`
- `milestone`

## 六、坚持天数规则

### 6.1 目标维度记录

坚持天数按**目标本身**记录，而不是全局记录。

示例：

- `每天阅读30分钟` 连续完成 5 天，则该目标 streak 为 5
- `晚上 11 点前睡觉` 是另一条独立 streak

### 6.2 中断规则

这里明确 streak 只围绕“今天承诺过的目标”计算：

- 某目标 **今天加入了今日承诺且未完成**，则该目标 streak 断开
- 某目标 **今天没有加入今日承诺**，则 streak 冻结，不增长也不断开
- 某目标 **今天加入了承诺且完成**，则 streak 增长
- 断开后下一次完成时从 1 重新开始

### 6.3 自定义目标规则

自定义目标也参与 streak 计算，但遵守以下规则：

- 同一文案连续保留并完成，可累计 streak
- 若文案被修改为新目标，则视为新目标，重新计算 streak

这保证“早睡打卡”和“晚上不刷手机”不会共用一条坚持记录。

## 七、数据结构调整

### 7.1 目标对象新增字段

在现有目标对象基础上补充：

```javascript
{
  id: string,
  name: string,
  type: 'habit' | 'milestone' | 'oneTime',
  icon: string,
  tag: string,
  createdFrom: 'recommended' | 'custom',
  baseXp: number,
  streakDays: number,
  bestStreakDays: number,
  lastCompletedDate: string | null,
  completed: boolean,
  // milestone 目标保留
  totalProgress: number | null,
  currentProgress: number,
}
```

### 7.2 向后兼容

本次**不做旧字段兼容设计**，统一按新模型实现：

- `baseXp` 是唯一基础经验字段
- `goal.xp` 不再作为核心数据来源
- 所有展示、心愿生成、结算逻辑统一改读新模型
- 若旧调用链引用 `goal.xp`，直接改造为新接口

### 7.3 展示字段与结算字段分离

建议将“目标静态基础值”和“完成时实时计算值”分开：

- `baseXp` 负责定义基础经验
- `getGoalPreviewXp()` 或等价逻辑负责返回当前展示用预计经验
- `calculateGoalXp()` 负责返回完成瞬间的实际结算经验

这样可以避免把实时计算结果直接写回目标定义。

### 7.4 重复目标策略

目标库中若出现同名目标，本次不追求保留旧实现习惯，而是明确为：

- 默认复用同名且未完成的目标
- 不重复创建新的同名目标对象
- 自定义目标改名后按“新目标”创建，不复用旧 streak

## 八、异常与边界处理

### 8.1 承诺槽位已满

- 今日承诺达到 4 个后，不允许继续添加
- 给出轻提示：`今天先完成这 4 个吧`

### 8.2 自定义输入取消

- 用户取消输入时直接关闭流程
- 不创建空目标，不污染目标库

### 8.3 无效输入

- 自定义目标为空、全空格或超长时不保存
- 给出温和提示，允许重新输入

### 8.4 重复目标

- 如果目标库已存在相同名称且尚未完成，优先复用已有目标
- 避免同名目标被重复创建多条

### 8.5 经验值异常兜底

- 若计算结果为 `NaN` 或非法值，则回退为 `baseXp`
