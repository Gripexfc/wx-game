# 下版本需求方向（C + E 双优先）

**日期：** 2026-04-23  
**状态：** 已定稿（指标优先级与主线范围）  
**关联文档：** `2026-04-22-好友激励与四宠物首启设计-design.md`、`2026-04-22-宠物背景升级与任务区留白利用-design.md`、`2026-04-21-目标选择与动态经验设计.md`

## 1. 背景

产品核心循环为「个人承诺 → 完成反馈 → 养成成长」。近期规格已覆盖：目标选择与动态经验、宠物背景语义化、好友激励（单向关注、分享/好友位、加油、来访双宠等）。  
**下版本在业务上同时追求 C 与 E**，即不将社交与留存对立，而以「轻关系链 + 可理解的个人坚持」共同抬升 7 日内回访与传播转化。

---

## 2. 北极星与版本目标

### 2.1 北极星（与好友激励 Spec 一致）

- 人均每周「好友激励触达次数」：查看好友进度、加油、与来访相关的高价值曝光（定义见好友激励 Spec）。

### 2.2 本版双优先指标（用户已确认：两个都要）

| 代号 | 指标 | 本版期望 |
|------|------|----------|
| **C** | 7 日留存（D7） | 关注/使用好友能力的用户组 D7 **高于**未使用组；若尚无线上埋点，以**体验走查 + 云库互动流水 + 抽样问卷**作心证 |
| **E** | 社交传播 | 分享带来打开、带参页到关注/首启的转化；**分享回流率**、**关注转化率**、**好友互动率**与好友激励 Spec 中 MVP 阈值对齐或作为努力方向 |

### 2.3 非目标（本方向文档不单独扩 scope）

- 不替代各子 Spec 中的技术细节；实现以各子 Spec 为准。  
- 不接全量数据埋点 SDK 仍可为团队决策，见好友激励 Spec 第 11 节；若后续接数，再单独立项。

---

## 3. 战略：一条主线 + 一条底托

1. **主线（服务 E，并间接服务 C）**  
   落地并跑通 **好友激励 MVP**（四宠首启、单向关注、公开摘要、每日加油、分享/好友位进入、来访双宠互动、云侧发奖与频控）。  
   增长闭环：**完成可分享 → 回流 → 关注沉淀 → 每日动态与加油 → 回访**。

2. **底托（服务 C，避免「只有空关系」）**  
   在已有 **目标与坚持、心情与经验、首页承诺** 体验上保持可用、可理解；不把 D7 完全押在「对方是否活跃」上。  
   与《目标选择与动态经验》一致：**个人坚持与可解释 XP** 仍是回流基础面。

---

## 4. 版本范围

### 4.1 必做（与好友激励 Spec V1 对齐）

- 首启：四宠物选一、起名、首目标（与四宠首启 Spec 一致）。  
- 社交：单向关注、好友动态/摘要、每日加油、来访与双宠互动演出、分享带参与落地关注路径。  
- 安全与体验：奖励仅云函数发放、频控与幂等、轻嘲与未成年的策略与开关（细节以好友激励 Spec 第 8、15 节为准）。  
- **发布前关单**好友激励 Spec 第 15.11 节 OPEN 项：**Q4 首访轻奖**、**Q5 未成年/轻嘲**、**Q6 share_links 方案**、**Q7 访客列表是否可关**——须定稿或明确列为「已知问题」与降级策略。

### 4.2 与体验底托相关的必做（跨 Spec 协调，不重复造需求）

- **目标与 XP**：按《目标选择与动态经验》执行，保证完成反馈可解释、streak 规则清晰。  
- **背景与陪伴感**：按《宠物背景语义化》阶段交付，避免任务完成后「无记忆点」场景。  
- 若子 Spec 之间有冲突，以**更早上线、风险更低**的一方为准，并在实现 PR 中注明。

### 4.3 选做增强（本版 Spec 不默认包含；择一需另开修订或专项）

以下**三选一**曾在头脑风暴中提出，**本文件不默认纳入本版本交付**，避免在 C+E 双押下范围失控；若你确认要做其中一项，请更新本文件或新增 Delta Spec：

- 周进度小结卡（单人回顾，偏 C）  
- 被加油/被来访的「收件箱/小红点」聚合（C+E 交界）  
- 关注列表强调「今天谁坚持」（偏 E + 回访动机）

**默认：本版不做以上选做；仅必做 4.1 + 协调 4.2。**

## 5. 验收与风险（摘要）

- **功能验收**：以好友激励 Spec 第 10 节、第 15.10 节清单为准；各子系统以各自 Spec 的验收节为准。  
- **C 的心证**：对比「有关注或稳定使用好友流」与「无」的 D7（或线下剧本满意度）；无埋点则不强求统计显著。  
- **E 的心证**：分享→打开→关注/首启链路可走通；互动率、回流率在团队可观察范围内向 Spec 目标靠拢。  
- **风险**：社交与云开发联调占工期；弱网与合规需在真机与体验版上复测。
