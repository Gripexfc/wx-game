# 《人生拾光》微信小游戏设计文档

## 1. 产品定位

**游戏名称：** 人生拾光
**类型：** 休闲收集·三消合成
**核心用户：** 18-40 岁年轻职场人、学生
**产品口号：** "拾光筑梦，不负韶华 —— 每一步成长，都值得被珍藏"
**核心价值：** 传递积极正向价值观，缓解焦虑，碎片化体验，无压力治愈

---

## 2. 视觉设计规范

### 2.1 配色方案

| 用途 | 颜色 | 色值 |
|------|------|------|
| 主色（暖阳） | 暖黄 | #F5B041 |
| 辅助色（希望） | 浅绿 | #7DCEA0 |
| 点缀色（温暖） | 淡蓝 | #85C1E9 |
| 背景色 | 米白渐变 | #FFF9E6 → #FFFFFF |
| 强调色（达成） | 珊瑚橙 | #F0A500 |
| 文字主色 | 深棕 | #5D4E37 |
| 文字辅色 | 浅棕 | #8B7355 |

### 2.2 美术风格

- **整体基调：** 年轻化温暖治愈风，圆角卡片设计，轻盈留白
- **字体：** 圆润无衬线字体，清晰易读
- **图标：** 扁平化 + 微渐变，卡通手绘感
- **碎片设计：** 5 种时光碎片，对应成长历程（童趣蓝、求知绿、青春黄、温情粉、圆梦金）
- **粒子效果：** 消除时轻量粒子飘散，柔和不刺眼
- **禁止元素：** 灰暗色调、负面表情、沉重氛围

### 2.3 界面设计原则

- **简洁留白：** 大量留白，信息层次分明，不超过 3 层视觉层级
- **圆角设计：** 卡片、按钮统一圆角（16-24px），柔和亲和
- **微动效：** 按钮点击缩放、碎片消除弹跳、界面切换滑入，流畅自然
- **单手操作：** 核心操作区在屏幕下半部分，适配单手握持

---

## 3. 核心玩法设计

### 3.1 三消合成机制

**棋盘规格：** 6×6 格子
**碎片类型：** 5 种时光碎片（幼年·求知绿、少年·活力橙、青年·奋斗蓝、中年·温情粉、成就·圆梦金）
**操作方式：** 滑动交换相邻碎片，匹配 3 个及以上相同颜色即消除
**合成规则：** 消除碎片随机掉落新碎片，有概率直接掉落高级碎片

### 3.2 关卡流程（12 关）

| 阶段 | 关卡 | 主题 | 核心任务 |
|------|------|------|---------|
| 童年启蒙 | 1-3 | 童趣、启蒙、友谊 | 收集对应碎片，达成目标分数 |
| 成长突破 | 4-6 | 求知、拼搏、收获 | 收集高级碎片，完成特定消除次数 |
| 温情相守 | 7-9 | 陪伴、感恩、回馈 | 消除指定位置碎片，传递温情主题 |
| 圆满收获 | 10-12 | 圆梦、成就、回顾 | 综合挑战，解锁全部成就 |

**单关时长：** 1-3 分钟
**难度曲线：** 前期低门槛 + 正向激励，中期适度挑战，后期综合验收

### 3.3 正向反馈机制

- 每消除成功：轻量粒子 + 叮声音效 + "太棒了" 文字
- 过关成功：全屏金色光效 + 碎片飞入图鉴 + 弹窗庆祝
- 失败处理：无惩罚，仅提示 "再试一次，你一定可以"
- 激励视频：用户主动触发双倍奖励，无强制

## 4. 功能系统

### 4.1 时光收集系统

- 每关结算后获得该关主题碎片
- 碎片存入成长图鉴，可随时查看
- 收集全部碎片解锁 "圆满逐光者" 称号

### 4.2 成就系统

| 成就名称 | 解锁条件 | 奖励 |
|---------|---------|------|
| 启蒙者 | 通关阶段一（关卡1-3） | 专属图鉴框 |
| 成长达人 | 通关阶段二（关卡4-6） | 背景皮肤 |
| 幸福守护者 | 通关阶段三（关卡7-9） | 特效动画 |
| 圆满逐光者 | 通关全部关卡 | 终极称号 + 成长报告 |

### 4.3 成长报告系统

- 全部通关后生成专属成长报告
- 展示：收集碎片总数、解锁成就、关卡通关时间
- 支持自愿分享至微信好友/朋友圈

### 4.4 激励奖励系统

- 首次通关：基础奖励
- 观看激励视频：双倍碎片奖励（用户主动触发）
- 每日登录：随机碎片奖励

## 5. 界面结构

### 5.1 界面列表

1. **启动页** — 游戏 Logo + 标语，淡入过渡
2. **主菜单** — 开始游戏、关卡选择、成长图鉴、设置（音效/关于）
3. **关卡选择** — 4 阶段 × 3 关，卡片式解锁状态展示
4. **游戏视图** — 棋盘（中央）、步数/目标（顶部）、暂停按钮（右上）
5. **结算弹窗** — 获得碎片展示、成就解锁提示、激励视频按钮
6. **成长图鉴** — 已收集碎片画廊、成就墙
7. **成长报告** — 通关后生成，支持分享

### 5.2 导航流程

```
启动页 → 主菜单
           ├→ 关卡选择 → 游戏视图 → 结算 → (主菜单/重玩/下一关)
           ├→ 成长图鉴
           └→ 设置
```

## 6. 技术方案

### 6.1 技术栈

- **渲染：** 微信原生 Canvas API
- **语言：** JavaScript（ES6+）
- **适配：** 微信小游戏适配层（wx API）
- **分包：** 基础包 + 资源包分离，减少首载体积

### 6.2 项目结构

├── systems/

### 6.3 数据持久化

- **方案：** wx.getStorageSync 本地存储
- **存储内容：** 进度（当前关卡）、碎片收集状态、成就解锁状态、设置（音效开关）
- **数据结构：**
  ```json
  {
    "progress": { "currentLevel": 1, "unlockedLevels": [1] },
    "fragments": { "green": 5, "orange": 3, "blue": 2, "pink": 1, "gold": 0 },
    "achievements": { "启蒙者": false, "成长达人": false, "幸福守护者": false, "圆满逐光者": false },
    "settings": { "sound": true, "music": true }
  }
  ```

### 6.4 广告接入（合规）

| 广告类型 | 触发场景 | 规则 |
|---------|---------|------|
| 激励视频 | 双倍奖励领取 | 用户主动点击，不强制 |
| 插屏广告 | 关卡结算后 | 间隔 ≥ 60 秒可触发 |

---

## 7. 合规要点

- 无虚拟支付、无内购
- 接入微信官方流量主广告
- 适配未成年人防沉迷（12+）
- 所有素材原创或授权
- 无诱导分享、无虚假宣传
- 适龄提示：12+

---

## 8. 开发优先级

**P0（核心体验）：** 游戏主循环、三消消除、关卡配置驱动、存档系统
**P1（完整功能）：** 主菜单、关卡选择、结算弹窗、成就系统、图鉴系统
**P2（商业化）：** 广告接入、激励视频
**P3（扩展）：** 成长报告、分享功能、关卡扩展
