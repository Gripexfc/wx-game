# 宠物背景升级与任务区留白利用设计

## 1. 背景与目标

当前首页宠物卡区域在信息结构上已经清晰，但宠物背景视觉层次偏单调；同时任务模块调整为正常排版后，会出现一段可利用的留白区域。  
本设计目标：

1. 为宠物卡提供更有层次的动态背景（治愈风 + 成长感 + 互动反馈）。
2. 在不打断主流程的前提下，利用任务区留白增加轻量信息价值。
3. 保持当前业务逻辑稳定，不引入高风险耦合。

已确认的设计方向：

- 风格：温馨治愈（A）+ 成长感（B）+ 互动感（C）
- 动效强度：中等（C）
- 优先级：视觉效果优先（A）
- 留白方案：版本 A（轻信息条）

## 2. 设计原则

1. **宠物主角优先**：背景永远服务主体，不喧宾夺主。
2. **动效可感知但不疲劳**：常态慢节奏，交互短促明确。
3. **成长可解释**：不同等级背景复杂度有明确分层。
4. **低侵入集成**：尽量复用现有 `HomePage` 渲染管线，不改核心业务协议。
5. **可降级**：保留后续接入性能档位（`motionTier`）的空间。

## 3. 方案对比与选型

### 方案 1：暖光云层（采用）

- 组成：奶油渐变底 + 柔光云团 + 主光池 + 漂浮粒子 + 交互特效层。
- 优点：和现有暖色 UI 一致，能同时承载治愈、成长、互动三目标。
- 风险：层数增加后有过绘风险，需要粒子与特效上限。

### 方案 2：小场景舞台（不采用）

- 组成：具象小场景（房间/花园）+ 装饰物升级。
- 优点：记忆点强。
- 不采用原因：会偏离当前抽象治愈风，改造成本高。

### 方案 3：几何光效（不采用）

- 组成：抽象几何和轨迹光效。
- 优点：风格化强。
- 不采用原因：治愈感弱，品牌语义不如方案 1 稳定。

## 4. 视觉结构设计（宠物背景）

宠物卡背景采用 5 层，从后到前：

1. **基底层**  
   - 双向奶油渐变，消除平涂感。
2. **云团层**  
   - 2~4 片半透明柔云，慢速漂移。
3. **焦点光层**  
   - 宠物脚下主光池 + 背后背光，强化主体聚焦。
4. **粒子层**  
   - 低密度暖色粒子，缓慢漂浮并渐隐。
5. **交互特效层**  
   - 点击涟漪、任务完成脉冲、升级环。

### 等级分层规则

- `Lv1-5`：基底层 + 焦点光层
- `Lv6-10`：增加云团层
- `Lv11-15`：增加粒子层
- `Lv16+`：交互层强化（颜色与持续时间略增强）

## 5. 动效设计

### 常态动效

- 云层往返周期：6~10 秒
- 光晕呼吸周期：3~4 秒
- 粒子生成速率：约 1~2 个/秒
- 粒子生命周期：2~4 秒

### 交互动效

1. **点击宠物**
   - 触发中心涟漪（约 0.45s）
   - 粒子亮度短时提升，不额外暴增粒子

2. **完成任务**
   - 宠物背后暖色脉冲（约 0.6s）
   - 粒子密度短时 +30%（约 0.8s）后恢复

3. **升级事件**
   - 在既有升级特效基础上叠加背景金色环

### 防疲劳约束

- 不使用高频旋转和大幅位移
- 特效并发上限 2
- 粒子池硬上限（建议 24）

## 6. 留白利用设计（任务区版本 A）

在任务区上方或任务区头部空隙增加「今日状态条」，一行三段信息：

1. 今日完成：`x / y`
2. 连胜天数：`n 天`
3. 距离升级：`remaining XP`

### 设计要求

- 单行展示，弱视觉层级（不抢宠物焦点）
- 字体与配色沿用现有状态文案体系
- 超宽时优先截断提示文案，核心数字不截断

## 7. 组件与职责划分

## 7.1 HomePage（保留主调度）

- 负责布局与绘制顺序管理
- 新增（或内聚）两个渲染入口：
  - `drawPetBackground(ctx, rect, state)`
  - `drawDailyStatusStrip(ctx, rect, data)`

## 7.2 PetBackgroundRenderer（建议新增）

- 职责：纯渲染与动画状态推进，不处理业务判断
- 输入：`level`、交互事件、时间帧、动效档位
- 输出：Canvas 绘制结果

## 7.3 DailyStatusProvider（可先内联）

- 职责：聚合状态条展示数据
- 数据来源：
  - `growth`
  - `goalManager.getTodayCommitments()`
  - `goal.streakDays`

## 8. 数据流

1. 业务层更新（任务完成、升级等）产生事件。
2. `HomePage` 在渲染循环中读取业务状态与事件标记。
3. 背景渲染器基于状态与事件输出当前帧。
4. 状态条由数据聚合器计算并绘制。

说明：

- 动画运行时状态只驻留内存，不持久化。
- 不新增本地存储 schema，避免迁移成本。

## 9. 异常与边界处理

1. 无承诺任务时状态条展示 `0/0`，连胜为 `--`。
2. 粒子池达到上限时停止新增，仅更新存量。
3. 连续快速点击时，按并发上限丢弃最旧特效。
4. 在后续接入省电档位时，降级规则为：
   - 关部分粒子
   - 减少云层数量
   - 缩短交互特效时长

## 10. 测试与验收

## 10.1 功能验收

1. 背景层次明显增强，宠物仍为视觉中心。
2. 点击宠物、完成任务、升级三类事件特效可见且可区分。
3. 状态条数值与真实业务一致（完成数、连胜、XP）。

## 10.2 回归验收

1. 不影响任务完成/撤销逻辑。
2. 不影响升级与现有宠物动作逻辑。
3. 不引入新的 linter 报错。

## 10.3 性能观察项

1. 首页连续停留 3 分钟，无明显掉帧加剧。
2. 快速点击宠物 20 次后，无明显内存增长异常。

## 11. 实施边界（YAGNI）

本次不做以下扩展：

1. 不新增完整场景化背景（房间/花园）。
2. 不新增背景主题切换入口。
3. 不新增背景资源文件（全部先用 Canvas 程序化绘制）。
4. 不改动任务业务规则（仅展示层增强）。

## 12. 里程碑与交付

1. 背景渲染层接入（含等级分层）
2. 三类交互特效接入
3. 今日状态条接入
4. 回归测试与参数微调

交付完成标志：

- 视觉与交互达到本文验收标准；
- 业务无回归；
- 代码通过现有 lint/test 基线。
