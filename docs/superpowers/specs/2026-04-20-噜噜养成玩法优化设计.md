# 噜噜养成玩法优化设计文档

**日期：** 2026-04-20
**主题：** 宠物·任务·经验值·好感值联动性优化
**核心理念：** 噜噜见证你的努力，陪伴你遇见更好的自己

---

## 一、背景与目标

当前游戏存在以下问题：
- 宠物与任务系统联动薄弱，宠物仅为展示品
- 经验值/心情值/爱星三条系统各自独立，缺乏联动
- 缺乏长期留存驱动力，用户满级后系统失去意义
- 心愿单机制缺失，任务系统不足以支撑「完成自我目标」的核心诉求

**设计目标：**
- 让宠物从「展示品」变成「有反馈的伙伴」
- 任务系统以「用户自己想做的事」为核心，而非游戏预设
- 宠物全程见证、陪伴、激励用户的目标完成过程
- 建立真实的情感闭环：用户努力 → 宠物幸福；用户放弃 → 宠物难过

## 二、系统架构

### 2.1 核心系统关系

```
长期目标库（用户自设 / 噜噜推荐）
        ↓
今日承诺（每天最多4个，从目标库选择）
        ↓
完成承诺 → XP结算 + 心情↑ + 爱星↑ + 目标进度↑
        ↓
日终结算（次日打开时）
        ↓
表现分 → 心情值更新 → 宠物状态变化
        ↓
长期激励（爱星积累 → 解锁酷炫动作）
```

### 2.2 数据存储变更

**新增存储键：**

| 键名 | 内容 |
|---|---|
| `goal_data` | `{ goals: [...], todayCommitments: [...] }` |
| `wish_data` | `{ todayWishes: [...], yesterdayUnfinished: null }` |
| `pet_state_data` | `{ moodValue, consecutiveDays, lastSettleDate, totalDays, completedDays }` |

**修改存储键：**

| 键名 | 变更 |
|---|---|
| `lulu_data` | 新增 `unlockedActions: []` |

## 三、目标系统

### 3.1 目标类型

| 类型 | 说明 | 每日重置 | 有进度 |
|---|---|---|---|
| 习惯 | 每天都要做的事 | ✅ | ❌ |
| 里程碑 | 一个大目标分期完成 | ❌ | ✅ |
| 一次性 | 今天想做的一件事 | ✅（完成后消失） | ❌ |

### 3.2 目标数据结构

```javascript
{
  id: string,           // 唯一ID
  name: string,         // 目标名称（如「每天跑步30分钟」）
  type: 'habit' | 'milestone' | 'oneTime',
  xp: number,           // 奖励XP（5-30，用户自设）
  totalProgress: number|null, // 里程碑总目标量（如跑完50公里）
  currentProgress: number,   // 当前进度
  createdAt: string,   // 创建日期
  lastDoneAt: string|null,   // 上次完成日期
  remindCount: number,  // 已提醒次数（最多3次）
  completed: boolean,    // 是否已完成（一次性目标）
}
```

### 3.3 目标推荐引擎

**触发条件：** 目标库为空时自动展示推荐列表；用户可主动刷新推荐

**预设推荐池（按标签分类）：**

```
运动类：每天跑步30分钟 / 每天走10000步 / 每天做20个俯卧撑
学习类：每天阅读30分钟 / 每天背20个单词 / 每天学一个新知识点
健康类：每天22:00前睡觉 / 每天喝8杯水 / 每天冥想10分钟
社交类：每天给家人打电话 / 每天联系一个朋友 / 每周见一次朋友
生活类：每天整理房间 / 每天记账 / 每周做一次大扫除
```

**每日随机推3-5个，用户可直接添加或自定义。**

### 3.4 今日承诺

- 每天最多4个承诺槽位
- 用户从目标库拖入/点击选择今日承诺
- 已选目标在本日锁定，不可重复选
- 已完成承诺打勾显示
- 昨日未完成的承诺**不再自动出现**（用户需主动重新选择）

## 四、宠物情绪状态系统（5级）

### 4.1 情绪分级

| 状态 | 心情值 | 外观 | 行为 | 对话风格 |
|---|---|---|---|---|
| 🥺 沮丧 | 0-39 | 颜色暗淡，动作极慢，趴着 | 叹气、躺平、慢慢眨眼 | 「不想动...」「你都不陪我...」 |
| 😐 安静 | 40-64 | 正常颜色，正常动作 | 正常自动行为 | 「陪你一起」「慢慢来」「加油哦」 |
| 😊 开心 | 65-84 | 颜色鲜艳，动作稍快 | 轻快蹦跳、摇摆 | 「太棒了！」「你真厉害！」 |
| 😄 幸福 | 85-94 | 最鲜艳，轻微发光 | 撒星星、转圈 | 「今天的你会比昨天更好！」 |
| 🥰 超级幸福 | 95-100 | 发光+心形粒子 | 快乐跳跃+爱星光环 | 「超级爱你！你是最棒的！」 |

### 4.2 心情衰减曲线（断联惩罚）

| 断联天数 | 心情衰减 | 累计心情（起始68） |
|---|---|---|
| 0天（当天活跃） | 0 | 68 |
| 1天 | -3 | 65 |
| 2天 | -8 | 57 |
| 3天 | -15 | 52 |
| 5天 | -25 | 43 |
| 7天+ | -35（每多1天额外-5，封底20） | ≤33 |

### 4.3 心情影响XP倍率

| 心情区间 | XP倍率 | 说明 |
|---|---|---|
| ≥85 | ×1.2 | 心情高涨加成 |
| ≥60 | ×1.0 | 正常 |
| ≥40 | ×0.9 | 略低 |
| <40 | ×0.8 | 心情低迷惩罚 |

单次任务XP上限：**30 XP**（心情倍率只影响浮动范围，不突破上限）

## 五、宠物见证互动系统

### 5.1 核心见证关系

每个长期目标都与宠物绑定，宠物感知并产生反应：

| 场景 | 宠物反应 | 对话 |
|---|---|---|
| 用户新建长期目标 | 开心、Hop | 「你有了新目标噜！我陪你一起！」 |
| 用户承诺今日目标 | 鼓励姿态 | 「说好了哦，今天噜噜会监督你！」 |
| 用户完成承诺 | 撒糖、超级开心 | 「做到了！你又进步了！」 |
| 用户漏做承诺 | 失落 | 「今天...没关系的，明天加油」 |
| 用户完成里程碑（25%/50%/75%/100%） | 专属庆祝 | 「太棒了！一半了！继续加油！」 |
| 用户放弃目标 | 难过 | 「这个不做了吗...没关系，有噜噜陪你」 |

### 5.2 长期目标进度提醒

| 进度 | 触发条件 | 对话 |
|---|---|---|
| 开始时 | 承诺目标时 | 「我记住了，我会关注你的进度」 |
| 停滞提醒 | 目标3天无进展 + 上线时30%概率 | 「那个『XX』目标，还在吗？我陪你」 |
| 进度加速 | 单日完成该目标2次+ | 「你在努力！我感觉到了！」 |
| 长期停滞 | 目标超过7天无进展 | 「『XX』目标，噜噜有点担心你...」 |
| 达成 | 进度=100% | 「你做到了！！噜噜好骄傲！！」 |

每个目标最多提醒3次，避免骚扰用户。

### 5.3 陪伴事件（随机触发）

| 类型 | 触发条件 | 对话示例 |
|---|---|---|
| 早安 | 用户回归（今日首次打开，且lastActiveDate≠今天） | 「早安！新的一天，我们一起加油~」 |
| 想念 | 断联≥1天，今日首次打开 | 「你终于来了！我等你好久了」 |
| 断联加深 | 断联≥3天 | 「...你是不是忘了我」（触发沮丧状态） |
| 鼓励 | 做任务时随机 | 「别急，慢慢来」「你已经很棒了！」 |
| 未完成心愿提醒 | 心愿未完成时，用户打开App | 「那个心愿...我还在等你完成呢」 |
| 加油 | 在线挂机时随机（每5分钟） | 「陪你在线！加油哦~」 |

### 5.4 连续打卡里程碑

**首次且唯一触发（每个里程碑只触发一次）：**

| 天数 | 触发条件 | 宠物反应 | 对话 |
|---|---|---|---|
| 3天 | 首次连续3天全完成 | 轻量鼓励 | 「3天了！你好厉害！」 |
| 7天 | 首次连续7天全完成 | 开心庆祝 | 「一周了！你是最棒的！」 |
| 14天 | 首次连续14天全完成 | 进阶庆祝 | 「两周了！噜噜见证了你的坚持！」 |
| 30天 | 首次连续30天全完成 | 特殊动画 | 「一个月了！遇见更好的自己噜！」 |

## 六、日终结算系统

### 6.1 结算触发时机

每次 `loadData` 时检测 `lastActiveDate`：
- 计算当前日期与 `lastActiveDate` 的差值（断开的天数）
- 对每一个完整断开的自然日，逐日执行结算
- 用户看到的是「最后一次活跃之后那个夜晚」的状态

### 6.2 单日结算逻辑

```javascript
// 1. 表现分计算
let score = 0;
for (each commitment) {
  if (completed) score += 15;  // 每个完成的任务+15分
}
score += consecutiveBonus; // 连签加分（见下）
score = Math.min(score, 100);

// 2. 连签加分（按梯度）
if (consecutiveDays >= 30) bonus = 12;
else if (consecutiveDays >= 15) bonus = 10;
else if (consecutiveDays >= 8) bonus = 8;
else if (consecutiveDays >= 4) bonus = 5;
else if (consecutiveDays >= 1) bonus = 3;
else bonus = 0;

// 3. 心情值更新
if (allTasksInComplete) {
  // 有任务承诺但全漏做 → 心情下降
  moodDelta = -8;
} else {
  // 有完成 → 心情上涨，上限+30
  moodDelta = Math.min(score * 0.3, 30);
}

// 4. 连签更新
if (todayCompleted >= 1) {
  consecutiveDays += 1;
} else {
  consecutiveDays = 0;
}

// 5. 里程碑完成度统计
totalDays += 1;
if (todayCompleted >= 1) completedDays += 1;

// 6. 宠物晚安状态记录
lastSettleDate = today;
petEveningMood = moodValue; // 记录入睡时的心情
```

### 6.3 早起状态（用户次日打开时）

```
检测 lastActiveDate：
  ├─ 如果 === 今天 → 正常继续（用户一直在）
  └─ 如果 !== 今天 → 补齐所有断开日的结算
                    → 显示早安语（根据昨晚petEveningMood生成）
                    → 宠物状态 = petEveningMood（反映昨晚入睡状态）
```

## 七、爱星系统与酷炫动作

### 7.1 爱星获取

| 来源 | 频率 | 产出 |
|---|---|---|
| 完成心愿任务 | 约2次/天 | 约2爱星/天 |
| 每次升级（levelUp） | 约9天/级 | 约1爱星/9天 |

保守估计：每月约60爱星，半年约360爱星。

### 7.2 解锁酷炫动作门槛

解锁条件：**半年内（180天）任务完成度 ≥ 门槛值**

| 档位 | 爱星消耗 | 半年完成度要求 | 说明 |
|---|---|---|---|
| 🥉 普通动作 | 80爱星 | ≥70% | 日常可触发 |
| 🥈 进阶动作 | 160爱星 | ≥80% | 里程碑时刻触发 |
| 🥇 终极动作 | 280爱星 | ≥90% | 终极成就触发 |

每档解锁1个动作（不重复），解锁后爱星消耗。

**半年完成度计算：**
```javascript
completionRate = (completedDays / totalDays) × 100%
```
存储在 `pet_state_data` 中，每次结算时更新 `totalDays` 和 `completedDays`。

### 7.3 酷炫动作设计（共9个）

**🥉 普通酷炫动作（80爱星，日常可触发）：**
- 「心跳突突」：胸前出现大爱心跳动3次
- 「蹦迪摇摆」：左右摇摆 + 星星从地面升起
- 「后空翻」：原地后空翻（心情≥65时才能触发）
- 「彩虹屁」：头顶出现彩虹 + 彩色粒子飘散
- 「原地起飞」：身体轻微上浮 + 周围光圈旋转

**🥈 进阶酷炫动作（160爱星，里程碑时刻触发）：**
- 「超级撒糖」：心形从四面八方飞向宠物 + 爱星环绕
- 「王者降临」：发光变亮 + 全场金色粒子 + 皇冠出现
- 「变身觉醒」：颜色切换为专属配色 + 光环特效

**🥇 终极酷炫动作（280爱星，终极成就触发）：**
- 「宇宙级庆祝」：全屏粒子 + 背景渐变 + 宠物体积短暂放大 + 字幕「噜噜永远陪着你」

**触发时机：**
- 普通动作：心愿完成 / 宠物等级提升时随机触发（30%概率）
- 进阶动作：连续7天全完成 / 达成里程碑进度时触发
- 终极动作：半年完成度≥90%时解锁，解锁瞬间全屏庆典

## 八、页面在线经验

- 玩家在宠物主页（HomePage）时，每 **1分钟** 自动奖励 **+1 XP**
- 无上限，在线越久涨越多
- 切到其他页面或退后台时停止计时
- 每分钟轻量Toast：「噜噜陪你涨了 +1 XP」（不打断操作）
- 在线XP正常触发levelUp和爱星奖励
